# Evidence README

## Purpose

Dieses Verzeichnis ist fuer verdichtete, datierte Evidence-Notizen vorgesehen,
wenn ein Task einen pruefbaren Claim, eine Entscheidung oder ein disqualifiziertes
Experiment festhalten muss.

Evidence ist:

- datiert
- claim-bezogen
- artefaktverknuepft
- knapp, aber belastbar

Evidence ist nicht:

- ein allgemeines Arbeitsjournal
- ein Dump von Rohlogs
- ein Ersatz fuer den Projekt-Report

## Naming

Evidence-Dateien folgen:

```text
docs/evidence/<track>/<workstream>/YYYY-MM-DD_<slug>.md
```

Beispiel:

```text
docs/evidence/models/svdpp/2026-04-12_ml1m_baseline_readout.md
```

## Minimal Content

Jede Evidence-Datei soll mindestens enthalten:

- Scope
- Claim oder Fragestellung
- Inputs und Artefakte
- Methode
- Readout
- Interpretation
- Decision oder Next Step
